Building & Setup

Building Fleet Credits Core

Prerequisites

üìã System Requirements: Ubuntu 20.04+, Debian 10+, or similar Linux distribution. macOS 10.14+ or Windows with WSL.

Required packages and libraries:

  • Compiler: GCC 7+ or Clang 5+
  • Build Tools: Autotools (autoconf, automake, libtool, autoconf-archive)
  • pkg-config: For dependency detection
  • Boost: 1.64+ (system, filesystem, thread, chrono, program_options, test)
  • OpenSSL: For cryptographic functions
  • Qt 5.x: For GUI (optional, if building fleetcredits-qt)
  • Berkeley DB: 4.8+ for wallet storage (libdb++-dev and libdb-dev)
  • Other: libevent-dev, libminiupnpc-dev, libzmq5-dev, python3, bsdmainutils
‚ö†Ô∏è Critical: autoconf-archive is required for configure script generation. Without it, you'll get: ./configure: line 16791: syntax error near unexpected token '11,'
‚ö†Ô∏è Critical: libdb++-dev is required for wallet functionality. Without it, you'll get: configure: error: libdb_cxx headers missing

Linux Build

Step 1: Install Dependencies

# Update system packages
sudo apt-get update
sudo apt-get upgrade -y

# Install build dependencies
sudo apt-get install -y \
  build-essential \
  libtool \
  autotools-dev \
  automake \
  pkg-config \
  bsdmainutils \
  python3 \
  libssl-dev \
  libevent-dev \
  libboost-system-dev \
  libboost-filesystem-dev \
  libboost-chrono-dev \
  libboost-test-dev \
  libboost-thread-dev \
  libboost-program-options-dev \
  libminiupnpc-dev \
  libzmq5-dev \
  libdb++-dev \
  libdb-dev \
  autoconf-archive \
  git \
  curl

# Install Qt dependencies (if building GUI)
sudo apt-get install -y \
  libqt5gui5 \
  libqt5core5a \
  libqt5dbus5 \
  qttools5-dev \
  qttools5-dev-tools \
  libprotobuf-dev \
  protobuf-compiler \
  libqrencode-dev

Step 2: Clone Repository

git clone https://github.com/JeremyGits/FleetCredits.git
cd FleetCredits

Step 3: Fix Permissions

# Make scripts executable (Git may not preserve permissions)
chmod +x autogen.sh genbuild.sh
chmod +x *.sh 2>/dev/null || true

Step 4: Generate Configure Script

./autogen.sh
Note: This may take 30-60 seconds. If you see "Permission denied", run chmod +x autogen.sh first.

Step 5: Configure Build

For a headless server (no GUI):

./configure \
  --with-gui=no \
  --enable-debug \
  --enable-tests \
  --enable-wallet \
  --with-miniupnpc \
  --enable-experimental-mweb \
  --enable-experimental-contributions

For full build with Qt GUI:

./configure \
  --enable-debug \
  --enable-tests \
  --enable-wallet \
  --with-miniupnpc \
  --enable-experimental-mweb \
  --enable-experimental-contributions

Step 6: Compile

# Build using all CPU cores
make -j$(nproc)

# Or build Qt client specifically
make qt/fleetcredits-qt
‚è±Ô∏è Build Time:
  • 4GB RAM, 2 CPU cores: ~20-30 minutes
  • 8GB RAM, 4 CPU cores: ~10-15 minutes
  • 16GB RAM, 8 CPU cores: ~5-10 minutes

macOS Build

# Install dependencies via Homebrew
brew install autoconf automake libtool pkg-config boost openssl berkeley-db4 qt5

# Build
./autogen.sh
./configure
make -j$(sysctl -n hw.ncpu)

Windows Build (Cross-Compilation)

Fleet Credits can be cross-compiled for Windows from Linux using WSL (Windows Subsystem for Linux) or a native Linux environment.

Prerequisites for Windows Cross-Compilation

  • WSL (Ubuntu/Debian): Recommended approach for Windows users
  • MinGW-w64: Cross-compilation toolchain
  • Autotools: Autoconf, automake, libtool
  • Dependencies: Same as Linux build, but cross-compiled

Automated Windows Build Script

Use the provided build_windows.sh script for automated cross-compilation:

# From WSL or Linux environment
cd fleetcredits
chmod +x build_windows.sh
./build_windows.sh

Manual Windows Build Steps

If building manually:

# Install MinGW-w64 toolchain
sudo apt-get install g++-mingw-w64-x86-64

# Build dependencies for Windows
cd depends
make HOST=x86_64-w64-mingw32

# Configure for Windows cross-compilation
cd ..
./autogen.sh
./configure --prefix=$(pwd)/depends/x86_64-w64-mingw32 \
            --host=x86_64-w64-mingw32 \
            --disable-tests \
            --disable-bench

# Build
make -j$(nproc)

# Binaries will be in src/ and src/qt/

Windows Build Output

After successful cross-compilation, binaries will be located in:

  • src/fleetcreditsd.exe - Core daemon
  • src/qt/fleetcredits-qt.exe - Qt GUI client
  • src/fleetcredits-cli.exe - Command-line interface
  • src/fleetcredits-tx.exe - Transaction utility

Required DLLs will also be built and should be copied alongside executables.

Common Build Issues

  • secp256k1 build-aux missing: Script automatically copies build-aux files
  • Missing dependencies: Ensure all depends are built before configuring
  • Permission errors: Some steps may require sudo for directory creation
  • Long build time: Cross-compilation can take 30+ minutes

Build Output Locations

After successful build, binaries are located in:

  • src/fleetcreditsd (Linux) or src/fleetcreditsd.exe (Windows) - Core daemon
  • src/qt/fleetcredits-qt (Linux) or src/qt/fleetcredits-qt.exe (Windows) - Qt GUI client
  • src/fleetcredits-cli (Linux) or src/fleetcredits-cli.exe (Windows) - Command-line interface
  • src/fleetcredits-tx (Linux) or src/fleetcredits-tx.exe (Windows) - Transaction utility

Verify Build Success

# Check binaries exist and have executable permissions
ls -lh src/fleetcreditsd
ls -lh src/fleetcredits-cli

# Check version
./src/fleetcreditsd -version

Troubleshooting

Build Issues

‚ùå Error: "Permission denied" when running autogen.sh 
chmod +x autogen.sh genbuild.sh
./autogen.sh
‚ùå Error: "./configure: line 16791: syntax error near unexpected token '11,'"

Solution: Install autoconf-archive package:

sudo apt-get install -y autoconf-archive
# Then re-run autogen and configure
./autogen.sh
./configure --with-gui=no --enable-debug --enable-tests --enable-wallet
‚ùå Error: "libdb_cxx headers missing"

Solution: Install Berkeley DB development package:

sudo apt-get install libdb++-dev libdb-dev

Then re-run configure:

./configure --with-gui=no --enable-debug --enable-tests --enable-wallet

Alternative: If you don't need wallet functionality, disable it:

./configure --with-gui=no --disable-wallet
‚ùå Error: "No rule to make target 'libunivalue.la'"

Solution: The univalue library needs to be built first. Run:

# First, make sure autoconf-archive is installed
sudo apt-get install -y autoconf-archive

# Build univalue library first
cd src/univalue
chmod +x autogen.sh
./autogen.sh
./configure
make
cd ../..

# Then build the main project
make -j$(nproc)

Alternative: Clean and rebuild everything from scratch:

make distclean
./autogen.sh
./configure --with-gui=no --enable-debug --enable-tests --enable-wallet
make -j$(nproc)
‚ùå Error: "No targets specified and no makefile found"

Solution: ./configure failed. Check for missing dependencies:

./configure --with-gui=no 2>&1 | grep -i "missing\|error"
‚ùå Error: "Could not link against boost_filesystem !"

Solution: Install Boost Filesystem development package:

sudo apt-get install libboost-filesystem-dev libboost-system-dev libboost-program-options-dev
sudo apt-get install libboost-all-dev  # Or install all Boost packages

If error persists, try:

# Clean previous configure attempts
make distclean 2>/dev/null || true
rm -f config.cache

# Re-run configure
./configure --with-gui=no --enable-debug --enable-tests --enable-wallet
‚ùå Error: "No rule to make target 'qt/fleetcredits-qt'"

Solution: Make sure you're building from the project root directory, not the src directory. The correct command is:

# From project root
make qt/fleetcredits-qt

# Or build everything
make -j$(nproc)

Running Fleet Credits

Initial Test Run

Test the daemon in regtest mode (local testing). Important: Regtest requires the -mocktime=1776643200 flag because the genesis block timestamp is in the future (April 20, 2026).

# Run daemon (Linux)
./src/fleetcreditsd -regtest -mocktime=1776643200 -printtoconsole

# Run daemon (Windows)
fleetcreditsd.exe -regtest -mocktime=1776643200 -printtoconsole

You should see initialization messages. Press Ctrl+C to stop.

Test RPC Commands

# Start daemon in background (Linux)
./src/fleetcreditsd -regtest -mocktime=1776643200 -daemon

# Start daemon in background (Windows)
fleetcreditsd.exe -regtest -mocktime=1776643200 -daemon

# Wait a few seconds, then test (Linux)
./src/fleetcredits-cli -regtest getblockchaininfo
./src/fleetcredits-cli -regtest getnetworkinfo
./src/fleetcredits-cli -regtest getblockhash 0

# Test (Windows)
fleetcredits-cli.exe -regtest getblockchaininfo
fleetcredits-cli.exe -regtest getnetworkinfo
fleetcredits-cli.exe -regtest getblockhash 0

The genesis block hash should be 556f8207d42c427a03d882d2e338a0102342648ce203f82a11efbe596da0f55e.

Run GUI Client

# Linux
./src/qt/fleetcredits-qt -regtest -mocktime=1776643200

# Windows
fleetcredits-qt.exe -regtest -mocktime=1776643200
‚ö†Ô∏è Genesis Block Mismatch: If you see a "genesis block mismatch" error, clear old regtest data: 
rm -rf ~/.fleetcredits/regtest/blocks
rm -rf ~/.fleetcredits/regtest/chainstate
rm -f ~/.fleetcredits/regtest/peers.dat
Then restart with -regtest -mocktime=1776643200.

Network Modes

  • Regtest: -regtest -mocktime=1776643200 - Local testing, instant blocks, isolated network. Requires mocktime flag.
  • Testnet: -testnet - Public test network (port 44556)
  • Mainnet: Default - Public production network (port 42069). Not yet launched (genesis timestamp is April 20, 2026).
Network Configuration: See Network Parameters for port numbers and configuration details. Current bootstrap peer: 96.126.123.195:42069 (mainnet). Testnet peers must be added manually with -addnode.
