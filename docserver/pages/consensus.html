Consensus & PoW

Proof-of-Work Consensus

Fleet Credits uses Scrypt-based Proof-of-Work for consensus.

Files: src/pow.cpp, src/fleetcredits.cpp, src/chainparams.cpp

Block Reward

Fixed 10,000 FC per block, no halving. See Reward System for contribution bonuses.

Difficulty Adjustment

Digishield algorithm adjusts difficulty every block to maintain 60-second block time.

Genesis Block Mining

The genesis block (block 0) is the foundation of the blockchain and must have valid Proof of Work.

Genesis Block Creation

During chain parameter initialization, the genesis block is created with:

  • Timestamp: January 1, 2026 00:00:00 UTC (1704067200)
  • Initial Nonce: 0
  • Difficulty: Network-specific (0x207fffff for regtest)
  • Block Reward: 10,000 FC
  • Coinbase Script: Contains genesis message "Nintondo"

Explicit PoW Mining

In chainparams.cpp, the genesis block is explicitly mined during initialization:

// Create genesis block
genesis = CreateGenesisBlock(1704067200, 0, 0x207fffff, 1, 10000 * COIN);

// Mine genesis block with valid PoW - CRITICAL for blockchain integrity
const Consensus::Params& consensusParams = consensus;
uint32_t maxNonceAttempts = 0xFFFFFFFF;
uint32_t nonceAttempts = 0;

while (!CheckProofOfWork(genesis.GetPoWHash(), genesis.nBits, consensusParams)) {
    ++genesis.nNonce;
    ++nonceAttempts;
    
    // Handle nonce wraparound
    if (genesis.nNonce == 0) {
        genesis.nTime++;
        genesis.nNonce = 0;
        nonceAttempts = 0;
        
        // Safety check: abort if mining fails
        if (genesis.nTime > 1704067300) {
            assert(false && "CRITICAL: Failed to mine genesis block");
        }
    }
    
    // Safety check: prevent infinite loops
    if (nonceAttempts >= maxNonceAttempts) {
        assert(false && "CRITICAL: Exceeded max nonce attempts");
    }
}

// Verify mined genesis block
assert(CheckProofOfWork(genesis.GetPoWHash(), genesis.nBits, consensusParams) &&
       "CRITICAL: Mined genesis block failed PoW validation");

// Store genesis hash
consensus.hashGenesisBlock = genesis.GetHash();

Why Explicit Mining?

The genesis block is mined during initialization to ensure:

  • Guaranteed Validity: Genesis block always passes PoW validation
  • Consistency: Same validation function used for mining and validation
  • Integrity: Blockchain integrity is guaranteed from block 0
  • No AuxPow Confusion: Genesis blocks don't have AuxPow, so validation falls back to standard PoW

Network-Specific Genesis Blocks

  • Mainnet: Pre-mined genesis block with specific hash
  • Testnet: Pre-mined genesis block with specific hash
  • Regtest: Genesis block mined dynamically during initialization with difficulty 0x207fffff (instant mining)

Genesis Block Validation

When validating blocks:

  1. CheckAuxPowProofOfWork() is called for all blocks
  2. Genesis blocks have no AuxPow, so it falls back to CheckProofOfWork()
  3. Since genesis block was mined with CheckProofOfWork(), validation always passes
  4. Checkpoint data stores genesis hash at height 0 for quick verification
Critical: The genesis block MUST have valid PoW. This is enforced during initialization, guaranteeing blockchain integrity from the very first block. The same CheckProofOfWork() function is used for both mining and validation, ensuring perfect consistency.
