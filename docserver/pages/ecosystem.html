Fleet Credits Ecosystem

Complete overview of the Fleet Credits blockchain ecosystem, components, and participants

What is the Fleet Credits Ecosystem?

The Fleet Credits ecosystem is a comprehensive blockchain platform that combines proven cryptocurrency technology with innovative features for privacy, contribution recognition, and community governance. Unlike traditional cryptocurrencies focused solely on value transfer, Fleet Credits creates a complete ecosystem that rewards diverse forms of value creation while maintaining optional privacy and decentralized governance.

üåç Ecosystem Philosophy: Fleet Credits is designed as a technical blockchain project that recognizes and rewards contributions to open-source development, education, creative work, and community value creation. The ecosystem enables a post-scarcity economic model where value derives from utility and impact, not speculation.

Core Infrastructure

Blockchain Layer

  • Consensus: Scrypt Proof-of-Work (ASIC-resistant, similar to Dogecoin)
  • Block Time: 60 seconds for fast transaction confirmation
  • Block Reward: Fixed 1.00000069 FC per block (no halving, unlimited issuance)
  • Network Isolation: Unique magic bytes and genesis block ensure complete separation from other blockchains
  • Difficulty Adjustment: Digishield algorithm for responsive difficulty retargeting

Privacy Layer: MWEB

Mimblewimble Extension Blocks (MWEB) provide optional transaction privacy:

  • Pedersen Commitments: Hide transaction amounts
  • Range Proofs: Verify amounts are non-negative without revealing values
  • Extension Blocks: Privacy transactions stored separately from transparent chain
  • Peg-in/Peg-out: Convert between transparent and private transactions
  • Dedicated Mempool: Separate mempool for MWEB transactions

See MWEB Overview for detailed information.

Governance System

Fleet Credits includes decentralized governance mechanisms:

  • Community Reserve: Funded from block rewards for ecosystem development
  • Proposal System: Submit governance proposals via RPC
  • Voting Mechanism: Community voting on proposals
  • Oracle Participation: Validators participate in contribution verification and governance

See Community Reserve for detailed information.

Ecosystem Participants

Miners

  • Role: Secure the network through Proof-of-Work
  • Rewards: Block rewards (1.00000069 FC base, up to 3.69428008 FC with contributions)
  • Requirements: Scrypt-capable mining hardware
  • Incentive: Higher rewards when including verified contributions in blocks

Contributors

  • Role: Create value through code, education, creative work, etc.
  • Rewards: Higher block rewards when contributions are verified and included
  • Process: Submit contributions via RPC, verified by oracles, included in blocks by miners
  • Types: Developers, educators, artists, mentors, AI validators, ethical reviewers

Validators/Oracles

  • Role: Verify contributions that require manual validation
  • Process: Review contribution proofs, vote on validity
  • Stakes: Oracles stake FC to participate in verification
  • Incentive: Fees from verified contributions, governance participation

Node Operators

  • Role: Run full nodes to maintain network infrastructure
  • Benefits: Direct access to blockchain data, RPC services, network participation
  • Requirements: Storage for blockchain data, network connectivity
  • Types: Seed nodes, RPC service providers, wallet nodes

Users

  • Role: Send and receive FC, use MWEB privacy features
  • Tools: Qt GUI wallet, CLI tools, third-party applications
  • Features: Transparent transactions, optional MWEB privacy, contribution tracking

Developers

  • Role: Build applications, tools, and services on Fleet Credits
  • Resources: RPC API, documentation, testnet/regtest environments
  • Opportunities: Wallet integrations, payment processors, contribution platforms, governance tools
