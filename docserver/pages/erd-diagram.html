ERD Diagram

Entity-Relationship Diagram

Complete ERD for Fleet Credits ecosystem.

File: fleetcredits/docs/ERD/globaldiagram.md

The ERD includes all entities: blocks, transactions, contributions, MWEB structures, oracles, validators, governance, and more.

Key Entities

  • Core Blockchain: CBlock, CBlockHeader, CTransaction, CTxIn, CTxOut, UTXO
  • Genesis Block: CBlock with hashPrevBlock=0, explicitly mined during initialization
  • Contribution System: CContributionTransaction, CProofData, 8 contribution types
  • MWEB Privacy: CMWEBExtensionBlock, CMWEBTransaction, Pedersen commitments, range proofs
  • Verification: COracleNode, CVerificationRecord, oracle consensus (3-of-5)
  • Rewards: BLOCK_SUBSIDY, BONUS_MULTIPLIER, REWARD_PAYOUT
  • Governance: CGovernanceProposal, CVote, CReserveFund

Genesis Block Mining Process

The ERD includes explicit modeling of genesis block mining:

  1. GENESIS_BLOCK_MINING entity creates genesis block
  2. Uses CreateGenesisBlock() and CheckProofOfWork()
  3. Increments nonce until PoW passes
  4. Stores genesis hash in CONSENSUS_PARAMS
  5. Adds checkpoint at height 0 via CCheckpointData
