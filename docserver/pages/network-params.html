Genesis Block Mining Process

For regtest networks, the genesis block is dynamically mined during chain parameter initialization:

  1. Create genesis block with initial parameters (timestamp, difficulty, reward)
  2. Mine block by incrementing nonce until CheckProofOfWork() passes
  3. Handle nonce wraparound by incrementing timestamp if needed
  4. Verify mined block passes PoW validation
  5. Store genesis hash in consensus parameters
  6. Add genesis hash (0c233e4e9a75...) to checkpoint data at height 0

Fleet Credits now hardcodes the mined genesis values in chainparams.cpp, eliminating the expensive "mine on startup" loop and reducing daemon boot time from minutes to seconds.

Configuration File

Edit ~/.fleetcredits/fleetcredits.conf (Linux/Mac) or %APPDATA%\\FleetCredits\\fleetcredits.conf (Windows):

# Mainnet configuration (fleetcredits.conf)
server=1
daemon=1
listen=1
bind=0.0.0.0
port=42069

# RPC (keep credentials secret)
rpcuser=YOUR_RPC_USERNAME
rpcpassword=YOUR_SECURE_PASSWORD
rpcport=42068
rpcallowip=127.0.0.1
# rpcallowip=TRUSTED_IP/32   # optional

# Recommended tuning
dbcache=1000
maxconnections=200
txindex=1
whitelist=TRUSTED_IP            # prevents bans for your management host
logtimestamps=1
debug=net
Chainstate Safety: If you previously ran a node pointed at another network (old ports or different genesis), delete blocks/ and chainstate/ in your data directory before restarting. Mixing data from incompatible chains triggers non-canonical ReadCompactSize() failures and automatic bans.

Network Isolation

Fleet Credits is completely isolated from Dogecoin and other networks:

  • Magic Bytes: Unique 4-byte identifiers prevent connecting to wrong networks
    • Mainnet: 0xFC 0x43 0x52 0x45 ("FCRE")
    • Testnet: 0xFC 0x54 0x45 0x53 ("FTES")
    • Regtest: 0xFA 0xBF 0xB5 0xDA (standard)
  • Genesis Block: Unique genesis block hash identifies Fleet Credits network
  • Seed Nodes: Initial bootstrapping uses 96.126.123.195:42069; additional seed entries can be compiled into chainparamsseeds.h as new public peers come online.
  • Port Numbers: Mainnet 42069 (P2P) / 42068 (RPC). Testnet/regtest retain legacy Dogecoin-compatible ports for tooling compatibility.

These ensure Fleet Credits nodes only connect to other Fleet Credits nodes and not Dogecoin or Bitcoin nodes.

Note: For development, use -regtest mode which is completely isolated. For testing, manually connect nodes with -addnode flags.
