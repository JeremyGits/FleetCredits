Components

Qt GUI Components

The Fleet Credits Qt GUI is built using Qt Framework, providing a cross-platform desktop application with a modern, user-friendly interface.

Directory: src/qt/

Main Files: fleetcreditsgui.h, fleetcreditsgui.cpp, fleetcredits.cpp

Main Components

1. FleetCreditsGUI

File: src/qt/fleetcreditsgui.h, src/qt/fleetcreditsgui.cpp

The main application window class, inheriting from QMainWindow. It manages:

  • Menu bar (File, Settings, Help)
  • Toolbar with quick actions
  • Status bar showing network status, sync progress, connection count
  • Tab-based interface for different views
  • Wallet management and switching
  • Connection to core client via signals/slots

2. WalletView

File: src/qt/walletview.h, src/qt/walletview.cpp

A stacked widget (QStackedWidget) that manages views for a single wallet:

  • Overview page
  • Send coins dialog
  • Receive coins dialog
  • Transaction view
  • Address book page
  • Import keys dialog

Each wallet instance has its own WalletView.

3. OverviewPage

File: src/qt/overviewpage.h, src/qt/overviewpage.cpp

The main dashboard showing:

  • Wallet balance (available, pending, total)
  • Recent transactions list
  • Transaction count
  • Sync status and progress

Uses TransactionTableModel and WalletModel for data.

4. SendCoinsDialog

File: src/qt/sendcoinsdialog.h, src/qt/sendcoinsdialog.cpp

Dialog for sending Fleet Credits:

  • Multiple recipient support via SendCoinsEntry
  • Amount input with validation
  • Fee calculation and display
  • Coin control (UTXO selection)
  • Transaction confirmation
  • Broadcasting via WalletModel

5. ReceiveCoinsDialog

File: src/qt/receivecoinsdialog.h, src/qt/receivecoinsdialog.cpp

Dialog for receiving Fleet Credits:

  • Generate new receiving addresses
  • Display QR codes for addresses
  • Request payment with amount labels
  • Recent requests list
  • Copy address/QR code functionality

Additional Dialogs

  • OptionsDialog - Settings and preferences
  • TransactionDescDialog - Transaction details
  • AddressBookPage - Address book management
  • CoinControlDialog - UTXO selection for transactions
  • SignVerifyMessageDialog - Message signing/verification
  • HelpMessageDialog - About and help information
  • PaperWalletDialog - Paper wallet generation
  • ImportKeysDialog - Import private keys

UI Forms

Qt Designer UI files (`.ui`) are located in src/qt/forms/:

  • Automatically generated header files in src/qt/forms/ui_*.h
  • Design-time UI definitions for all dialogs and pages
  • Compiled using Qt's uic tool

Component Architecture

The GUI follows Qt's Model-View architecture:

  • Models: Provide data (WalletModel, ClientModel, etc.)
  • Views: Display data (QTableView, QListView, etc.)
  • Dialogs: User interaction (SendCoinsDialog, etc.)
  • Delegates: Custom rendering (TxViewDelegate, etc.)

Signal/Slot Communication

Components communicate via Qt's signal/slot mechanism:

  • Models emit signals when data changes
  • Views connect to model signals for updates
  • Core client events trigger GUI updates
  • User actions trigger RPC calls or wallet operations
