Models

Qt Models

Qt models provide data to views using the Model-View architecture. Models inherit from Qt's QAbstractItemModel or specialized subclasses.

Directory: src/qt/

Core Models

1. ClientModel

File: src/qt/clientmodel.h, src/qt/clientmodel.cpp

Provides core client data to the GUI:

  • Network Status: Connected peers, network state
  • Blockchain Info: Block count, chain height, sync progress
  • Consensus: Current difficulty, hash rate
  • Version: Client version, protocol version
  • Notifications: Block tip updates, alerts

Emits signals when core client state changes, allowing GUI components to update automatically.

2. WalletModel

File: src/qt/walletmodel.h, src/qt/walletmodel.cpp

The main wallet interface for the GUI, providing:

  • Balance: Available, pending, total balances
  • Transactions: Transaction history and status
  • Addresses: Address generation and management
  • Send Operations: Creating and broadcasting transactions
  • Encryption: Wallet encryption/decryption status
  • Options: Wallet-specific settings

Acts as a bridge between CWallet (core) and GUI components, emitting signals for balance changes, new transactions, etc.

3. TransactionTableModel

File: src/qt/transactiontablemodel.h, src/qt/transactiontablemodel.cpp

Table model for displaying transactions in views:

  • Columns: Status, Date, Type, Address, Amount, etc.
  • Sorting: Sortable by any column
  • Formatting: Formats amounts, dates, addresses for display
  • Updates: Automatically updates when wallet transactions change

Used by OverviewPage and TransactionView to display transaction lists.

4. AddressTableModel

File: src/qt/addresstablemodel.h, src/qt/addresstablemodel.cpp

Table model for the address book:

  • Columns: Label, Address, Type (receiving/sending)
  • Operations: Add, edit, remove addresses
  • Types: Receiving addresses (for payments) and sending addresses (from transactions)
  • Updates: Updates when address book changes

Additional Models

5. RecentRequestsTableModel

File: src/qt/recentrequeststablemodel.h, src/qt/recentrequeststablemodel.cpp

Model for payment requests:

  • Displays recent payment requests
  • Stores request details (label, amount, date)
  • Used in ReceiveCoinsDialog

6. PeerTableModel

File: src/qt/peertablemodel.h, src/qt/peertablemodel.cpp

Table model for peer connections:

  • Displays connected peers
  • Shows peer address, version, ping time
  • Used in debug window

7. OptionsModel

File: src/qt/optionsmodel.h, src/qt/optionsmodel.cpp

Model for application settings:

  • Displays current settings
  • Handles setting changes
  • Persists settings to disk

8. TransactionFilterProxy

File: src/qt/transactionfilterproxy.h, src/qt/transactionfilterproxy.cpp

Proxy model for filtering and sorting transactions:

  • Filters transactions by type, date range, amount
  • Custom sorting options
  • Sits between TransactionTableModel and views

Model Updates

Models update automatically when underlying data changes:

  • WalletModel: Updates when wallet balance or transactions change
  • ClientModel: Updates when blockchain or network state changes
  • Table Models: Update when wallet or address book changes

Updates are thread-safe and use Qt's signal/slot mechanism to notify views.
