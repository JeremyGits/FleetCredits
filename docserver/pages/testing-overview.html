Testing Overview

Testing Fleet Credits

Fleet Credits includes comprehensive test suites covering all major components, from cryptographic primitives to blockchain validation and contribution systems.

Test Directory: src/test/

Test Framework

Fleet Credits uses Boost.Test for unit testing, providing:

  • Test fixtures and setup/teardown
  • Assertion macros (BOOST_CHECK, BOOST_REQUIRE, etc.)
  • Test suites and cases
  • Mock objects and test utilities

Test Types

1. Unit Tests

Individual component tests:

  • test_fleetcredits.cpp - Main test suite
  • fleetcredits_tests.cpp - Fleet Credits-specific tests
  • crypto_tests.cpp - Cryptographic function tests
  • hash_tests.cpp - Hash function tests
  • scrypt_tests.cpp - Scrypt PoW algorithm tests
  • key_tests.cpp - Key generation and signing tests
  • amount_tests.cpp - Amount calculation tests
  • arith_uint256_tests.cpp - 256-bit arithmetic tests
  • serialize_tests.cpp - Serialization/deserialization tests

2. Blockchain Tests

Blockchain and consensus validation:

  • main_tests.cpp - Main blockchain validation tests
  • pow_tests.cpp - Proof-of-Work difficulty tests
  • auxpow_tests.cpp - Auxiliary Proof-of-Work tests
  • merkle_tests.cpp - Merkle tree tests
  • blockencodings_tests.cpp - Block encoding tests
  • coins_tests.cpp - UTXO set tests
  • mempool_tests.cpp - Memory pool tests

3. Transaction Tests

Transaction creation and validation:

  • transaction_tests.cpp - Transaction structure tests
  • script_tests.cpp - Script execution tests
  • script_P2SH_tests.cpp - Pay-to-Script-Hash tests
  • multisig_tests.cpp - Multisignature tests
  • sighash_tests.cpp - Signature hash tests

4. MWEB Tests

Mimblewimble Extension Blocks:

  • mweb_tests.cpp - MWEB structure and operations tests
  • mweb_test.cpp - Additional MWEB functionality tests

5. Wallet Tests

Wallet functionality (in src/wallet/test/):

  • wallet_tests.cpp - Core wallet operations
  • crypto_tests.cpp - Wallet cryptography

6. Network Tests

Network and peer management:

  • net_tests.cpp - Network protocol tests
  • netbase_tests.cpp - Network address tests
  • addrman_tests.cpp - Address manager tests

7. RPC Tests

RPC command tests:

  • rpc_tests.cpp - RPC command execution tests

Running Tests

Run All Tests

cd fleetcredits
make check
# or
./src/test/test_fleetcredits

Run Specific Test Suite

./src/test/test_fleetcredits --run_test=fleetcredits_tests
./src/test/test_fleetcredits --run_test=mweb_tests

Run with Verbose Output

./src/test/test_fleetcredits --log_level=all

Fuzzy Testing

Fleet Credits includes fuzzy testing for deserialization robustness:

  • test_fleetcredits_fuzzy.cpp - Fuzz tests for deserialization

Fuzzy tests help ensure the client handles malformed or malicious input gracefully.

Test Utilities

Test helper functions are available in:

  • testutil.h / testutil.cpp - Common test utilities
  • test_fleetcredits.h - Test suite headers

Continuous Integration

Tests should be run before:

  • Submitting pull requests
  • Creating releases
  • Making significant code changes

Writing New Tests

When adding new features, include corresponding tests:

  1. Create test file in src/test/ or appropriate subdirectory
  2. Use Boost.Test macros for assertions
  3. Include test in main test suite
  4. Run tests locally before committing

Regtest Mode for Testing

Fleet Credits supports regtest mode for isolated local testing. Important: Regtest requires the -mocktime=1776643200 flag because the genesis block timestamp is in the future (April 20, 2026).

Starting Regtest

# Daemon
./src/fleetcreditsd -regtest -mocktime=1776643200 -daemon

# Qt Client
./src/qt/fleetcredits-qt -regtest -mocktime=1776643200

# Verify genesis block
./src/fleetcredits-cli -regtest getblockhash 0
# Should return: 556f8207d42c427a03d882d2e338a0102342648ce203f82a11efbe596da0f55e

Regtest Configuration

Create ~/.fleetcredits/regtest/fleetcredits.conf:

regtest=1
port=18444
rpcport=42068
addnode=YOUR_SERVER_IP:18444
Genesis Block Mismatch: If you encounter a "genesis block mismatch" error, clear old regtest data: 
rm -rf ~/.fleetcredits/regtest/blocks
rm -rf ~/.fleetcredits/regtest/chainstate
rm -f ~/.fleetcredits/regtest/peers.dat
Then restart with -regtest -mocktime=1776643200.
