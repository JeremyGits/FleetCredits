Verification System

Contribution Verification

Fleet Credits uses a multi-layered verification system combining automated checks, oracle consensus, and human validator pools to ensure contribution quality and prevent abuse.

Files: src/primitives/verification.h, src/primitives/verification.cpp, src/rpc/contribution.cpp

Verification Methods

1. Automated Verification

Certain contribution types can be automatically verified:

  • Code Contributions: GitHub API integration verifies commits, pull requests, and code quality metrics
  • AI Validation: Accuracy scores ‚â•0.8 automatically qualify for rewards
  • Data Labeling: Consistency checks against ground truth datasets

Automated verification provides instant validation and reduces oracle workload for high-volume, quantifiable contributions.

2. Oracle Consensus

For subjective or complex contributions, oracle nodes provide consensus-based verification:

  • Charitable Acts: Requires 3-of-5 oracle votes for verification
  • Ethical Reviews: Requires 3-of-5 oracle votes for verification
  • Educational Content: Quality assessment by oracle consensus
  • Creative Works: Community value assessment

Oracle consensus ensures decentralized verification while maintaining high standards for subjective contributions.

3. Human Validator Pools

Reputation-gated validator pools provide batch verification for:

  • High-volume contribution types
  • Community oversight and quality control
  • Cross-validation of oracle decisions

Validators must maintain reputation scores above thresholds and face slashing for incorrect verifications.

Oracle Nodes

Oracle nodes are trusted validators responsible for verifying contributions that require human judgment:

Requirements

  • Minimum Stake: 500,000 FC must be locked
  • Reputation: Must maintain positive reputation score
  • Availability: Must respond to verification requests within time limits

Selection Process

  1. Candidates submit stake and metadata
  2. Community governance votes on oracle selection
  3. Selected oracles enter active pool
  4. Performance is tracked and affects reputation

Slashing Mechanism

Oracles face slashing (loss of stake) for:

  • Incorrect verification votes (caught by consensus)
  • Missing too many verification requests
  • Collusion or malicious behavior

Verification Status

Contributions progress through verification states:

  • pending: Submitted but not yet verified
  • confirmed: Transaction confirmed in block (1+ confirmations)
  • verified: Successfully verified (12+ confirmations)
  • orphaned: Transaction included in an orphaned block

Verification Timing

Verification typically occurs:

  • Automated: Immediately upon submission
  • Oracle Consensus: Within 1-3 blocks (60-180 seconds)
  • Final Status: After 12 confirmations for reward eligibility

Future Enhancements

Planned verification improvements include:

  • GitHub API integration for automated code verification
  • AI accuracy scoring system integration
  • Reputation-weighted oracle voting
  • Automated slashing for detected fraud
