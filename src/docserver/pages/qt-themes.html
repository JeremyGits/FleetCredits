Themes

Qt GUI Themes

Fleet Credits supports multiple themes and color schemes, allowing users to customize the GUI appearance.

Files: src/qt/themebuilder.h, src/qt/themebuilder.cpp

Theme System

Themes are managed via the ThemeBuilder class and related components:

  • Theme Definition: Themes define colors, fonts, and styles
  • Dynamic Switching: Users can switch themes without restart
  • Custom Themes: Support for user-created custom themes
  • Preview: Theme preview widget for testing themes

Theme Components

ThemeBuilder

File: src/qt/themebuilder.h, src/qt/themebuilder.cpp

Dialog for creating and editing themes:

  • Color picker for theme colors
  • Preview widget showing theme appearance
  • Save/load theme definitions
  • Export themes for sharing

ThemePreviewWidget

Widget that displays a preview of how the theme looks:

  • Shows sample UI elements
  • Updates in real-time as colors change
  • Used in theme builder dialog

Default Themes

Fleet Credits includes several built-in themes:

  • Light Theme: Traditional light color scheme
  • Dark Theme: Dark mode for reduced eye strain
  • Classic Theme: Original Bitcoin-style appearance
  • Modern Theme: Contemporary design with gradients

Theme Colors

Themes define colors for:

  • Background: Main window and dialog backgrounds
  • Foreground: Text and icon colors
  • Accent: Highlight colors for buttons and links
  • Success/Warning/Error: Status indicator colors
  • Border: Border and separator colors

Theme Storage

Themes are stored:

  • Built-in Themes: Compiled into the application
  • Custom Themes: Stored in user data directory
  • Format: JSON or INI configuration files
  • Location: ~/.fleetcredits/themes/ (Linux) or equivalent

Applying Themes

Themes are applied:

  1. User selects theme from Settings dialog
  2. Theme data is loaded from storage
  3. Qt stylesheets are updated dynamically
  4. All GUI components refresh to new theme
  5. Theme preference is saved for next session

Style Sheets

Fleet Credits uses Qt Style Sheets (QSS) for theming:

  • CSS-like syntax for styling widgets
  • Dynamic style sheet generation from theme colors
  • Per-widget customization support
  • State-based styling (hover, pressed, disabled, etc.)

Future Enhancements

Planned theme system improvements:

  • Theme marketplace for sharing custom themes
  • Per-component theme customization
  • High contrast accessibility themes
  • Animated theme transitions
  • System theme integration (follow OS dark/light mode)
